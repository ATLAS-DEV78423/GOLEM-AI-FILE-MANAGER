# Operations

This document covers day-2 operations: installation, upgrades, common
issues, and recovery.

## Installation paths

| OS | Default install | Data dir |
|---|---|---|
| Windows | `%LOCALAPPDATA%\Programs\GOLEM` | `%LOCALAPPDATA%\GOLEM` |
| macOS | `/Applications/GOLEM.app` | `~/Library/Application Support/GOLEM` |
| Linux (dev only) | `<repo>` | `~/.golem` |

The data dir can be overridden with the `GOLEM_DATA_DIR` environment
variable.

## Verifying a fresh install (Windows)

```powershell
# 1. Install
.\GOLEM-2.0.0-windows-installer.exe

# 2. Verify the install dir
Test-Path "$env:LOCALAPPDATA\Programs\GOLEM\GOLEM.exe"

# 3. Verify the data dir was created
Test-Path "$env:LOCALAPPDATA\GOLEM\golem.db"

# 4. Verify the registry entry
Get-ItemProperty "HKCU:\Software\Microsoft\Windows\CurrentVersion\Uninstall\GOLEM" | Select DisplayName, DisplayVersion

# 5. Launch and accept the onboarding
& "$env:LOCALAPPDATA\Programs\GOLEM\GOLEM.exe"
```

## Verifying a fresh install (macOS)

```bash
# 1. Open the DMG
open GOLEM-2.0.0-macOS.dmg

# 2. Drag to Applications
cp -R /Volumes/GOLEM/GOLEM.app /Applications/

# 3. Launch
open /Applications/GOLEM.app

# 4. Grant Accessibility permission (for the global hotkey)
# System Settings -> Privacy & Security -> Accessibility -> enable GOLEM
```

## Common issues

### "Database is locked"

SQLite is in WAL mode with a 30s busy timeout. If you see this in
the log, two writers are competing for the same record. This should
not happen in the current code (single-writer design); if it does,
file a bug with the log attached.

### "Vault not found" / "Watched folder not found"

The user moved or deleted the folder. Open Settings from the tray
menu and re-pick.

### "API key rejected"

The provider rejected the key. Either it was entered wrong, the
account is out of credits, or the key has been revoked. Open
Settings, re-enter, and "Test" before saving.

### "Hotkey doesn't work"

Windows: `keyboard` requires admin privileges. If GOLEM couldn't
register the hotkey, it tries `pynput` automatically. If both fail,
the user can still open the search popup from the tray menu.

macOS: the user must grant Accessibility permission the first time.
System Settings -> Privacy & Security -> Accessibility -> enable
GOLEM, then relaunch.

### "I want to start over"

Tray menu -> "Reset all settings". This wipes the database
(`settings`, `files`, `files_fts`, `undo_log`) and re-opens
onboarding. Files in the watched folder and the vault are not
touched.

## Recovery

### Restore from a backup

```bash
# Stop GOLEM (right-click tray -> Quit)
cp backup/golem.db <data_dir>/golem.db
# Restart GOLEM
```

The DB is the canonical index. The vault and the `GOLEM Files/`
folder can be regenerated by re-running a scan; if the watched folder
still has the source files, just restart.

### Export the database for support

```bash
python main.py --export-db C:\backup\golem.db
```

Attach the exported DB to a bug report.

### View the log

- Tray menu -> "View log"
- Or: open `<data_dir>/golem.log` in a text editor

The log is rotated by overwriting (we do not keep old logs); copy it
elsewhere if you need history.

## Upgrades

A new release ships as:

- Windows: `GOLEM-<version>-windows-installer.exe` (a self-contained
  installer)
- macOS: `GOLEM-<version>-macOS.dmg`

Both replace the previous install in place. The data dir is
preserved. On first launch after upgrade, the app runs any pending
migrations (currently just the `groq_api_key` -> `llm_api_key`
promotion).

If you are upgrading from a v1.x install and your API key is missing
or your watched folder is not picked up, open Settings and re-enter
the values.

## Uninstall

Windows:

- Settings -> Apps -> Installed apps -> GOLEM -> Uninstall
- Or: Start Menu -> "Uninstall GOLEM"
- Or: `GOLEM-Setup.exe --uninstall --silent`

macOS:

- Drag `GOLEM.app` to the Trash
- (Optional) `rm -rf ~/Library/Application\ Support/GOLEM`

The uninstaller removes the install dir, the Start Menu and Desktop
shortcuts, and the registry entry. It does **not** touch the
Obsidian vault or the watched folder.
